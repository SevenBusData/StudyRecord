# Blog系统需求设计文档

# 一、需求总览

## 1.1 设计目标

打造一款「简洁高级、易用性强、可扩展性高」的个人/中小型博客系统，兼顾内容创作、阅读体验与管理效率，支持个性化定制，适配多端访问（PC端、移动端），满足用户“写得顺畅、看得舒适、管得便捷”的核心需求，同时预留功能扩展空间，适配后续内容沉淀、社交互动等延伸场景。

核心定位：以内容为核心，突出文字质感与视觉高级感，拒绝冗余功能，兼顾实用性与美观度，适配个人博主、垂直领域创作者（如技术、生活、好物分享）的使用场景。

## 1.2 核心价值

- 对创作者：提供简洁高效的编辑器，支持多格式排版、草稿保存、内容管理，降低创作门槛，助力内容沉淀与个人品牌打造；

- 对阅读者：提供无干扰、沉浸式的阅读体验，支持分类浏览、搜索、收藏，快速找到感兴趣的内容；

- 对管理者：提供直观的后台管理界面，支持内容审核、数据统计、用户管理，高效维护博客秩序与运营状态。

## 1.3 范围界定

### 1.3.1 纳入范围

- 前端：PC端博客首页、文章详情页、分类页、标签页、关于页、搜索页、登录/注册页、个人中心、编辑器页面；

- 后端：管理员账号管理（仅本人）、内容管理（文章、分类、标签）、评论管理、数据统计、权限控制（仅本人拥有管理员权限）、文件上传；

- 多端适配：PC端优先，移动端自适应（响应式设计）；

- 基础功能：管理员负责创作、编辑、发布、删除、修改、评论、收藏、分类、标签、个人资料设置；游客仅可浏览、搜索。

### 1.3.2 暂不纳入范围

- 复杂社交功能（如关注、私信、粉丝体系），后续可作为扩展功能；

- 付费阅读、广告植入功能；

- 多语言支持（优先中文，后续可扩展）；

- 高并发适配（针对中小型博客，暂不考虑万人同时在线场景）。

# 二、用户需求（User Requirement）

## 2.1 用户角色划分

系统核心分为2类角色，各角色权限与需求明确区分，避免权限混乱，同时保障使用体验；其中管理员仅本人可登录，保障系统安全性。

|角色|核心权限|核心需求|
|---|---|---|
|游客（未登录用户）|浏览文章、搜索文章、查看分类/标签、查看关于页、查看评论（不可评论）|无干扰阅读、快速找到感兴趣内容、界面简洁美观、加载流畅|
|管理员（仅本人）|所有权限 + 文章管理、分类/标签管理、评论审核/删除、个人资料管理、数据统计、系统设置（仅本人可操作）|高效管理内容、快速审核评论、了解阅读数据、便捷修改系统配置，保障账号安全|
## 2.2 核心用户场景

### 2.2.1 创作者场景（管理员）

- 场景1：日常创作发布文章——打开编辑器，输入内容，设置标题、分类、标签，插入图片/代码块，预览效果，保存草稿或直接发布；

- 场景2：修改已发布文章——找到对应文章，编辑内容、调整分类/标签，重新发布；

- 场景3：管理内容——查看所有文章，按状态（已发布、草稿、回收站）筛选，删除/恢复文章，批量操作；

- 场景4：管理评论——查看用户评论，审核违规评论，删除不当评论，回复用户评论；

- 场景5：查看数据——了解文章阅读量、评论量、收藏量，掌握用户偏好，优化创作方向。

### 2.2.2 阅读者场景（游客/普通用户）

- 场景1：浏览首页——查看最新发布的文章、热门文章，快速了解博主近期创作内容；

- 场景2：精准找内容——通过分类、标签筛选，或搜索关键词，找到感兴趣的文章；

- 场景3：阅读文章——沉浸式阅读，查看文章排版、图片、代码，查看评论区内容（不可评论）；

- 场景4：无额外互动功能（评论、收藏仅管理员可操作，贴合管理员仅本人的设定）。

# 三、功能需求（Functional Requirement）

## 3.1 公共功能（所有角色可见/可用）

### 3.1.1 首页功能

- 展示模块：顶部导航栏（logo、导航菜单、搜索框、登录/注册按钮）、banner区（可自定义图片/文字）、最新文章列表、热门文章（按阅读量排序）、分类导航、标签云、底部信息（关于博主、联系方式、版权信息）；

- 交互功能：导航菜单跳转、搜索框输入关键词搜索、分类/标签点击跳转、文章卡片点击进入详情页、页面滚动加载更多文章；

- 优化点：加载动画（避免白屏）、文章卡片hover微动效（提升高级感）、响应式适配（移动端隐藏部分模块，优化排版）。

### 3.1.2 文章详情页功能

- 展示模块：文章标题、发布时间、作者、阅读量、评论量、收藏量、分类/标签、文章内容（支持多格式排版）、相关推荐文章（同分类/同标签）、评论列表；

- 交互功能：管理员登录后可收藏/取消收藏、发表评论/回复评论、返回顶部、分享文章（复制链接、微信/微博分享）、代码块高亮/复制；游客仅可查看，无操作权限；

- 优化点：文章内容分段清晰、字体大小可调节、夜间模式切换（提升阅读体验）、图片点击放大查看、评论区分层显示（区分评论与回复）。

### 3.1.3 分类/标签页功能

- 分类页：展示所有分类，点击分类查看该分类下所有文章（按发布时间排序），显示分类描述、文章数量；

- 标签页：展示所有标签（标签云形式，字体大小区分热门程度），点击标签查看该标签下所有文章，支持标签搜索；

- 交互功能：分类/标签排序（按文章数量、创建时间）、文章列表分页/滚动加载。

### 3.1.4 搜索功能

- 搜索范围：文章标题、文章内容、标签；

- 交互功能：输入关键词实时联想（可选）、搜索结果分页展示、搜索无结果时提示相关推荐；

- 优化点：搜索速度快（避免卡顿）、关键词高亮显示。

### 3.1.5 关于页功能

- 展示内容：博主简介、创作理念、个人联系方式（如微信、公众号、Github等，可自定义显示/隐藏）、博客运营说明、站点统计（文章总数、运行时间等，可选）；

- 交互功能：管理员可登录后编辑关于页所有内容（修改简介、更新联系方式等），游客仅可查看，无编辑权限；

- 优化点：排版简洁高级，可插入个人头像、背景图，与博客整体风格统一。

## 3.2 管理员专属功能（仅本人可操作）

### 3.2.1 登录/注册功能（仅管理员可用）

- 登录功能：支持账号密码登录，可选添加验证码、记住密码（仅本人设备可用）、忘记密码重置功能（保障账号安全，仅本人可重置）；

- 注册功能：仅本人可完成注册（可设置关闭注册入口，避免他人注册），注册后自动获取全部管理员权限；

- 安全优化：登录失败多次后锁定账号（可自行解锁），密码加密存储，避免信息泄露。

### 3.2.2 个人中心功能

- 个人资料管理：编辑博主昵称、头像、简介、联系方式，修改登录密码，保存个人偏好设置（如编辑器主题、后台界面风格）；

- 创作管理入口：快速进入编辑器、文章管理、分类/标签管理页面，查看草稿、已发布文章；

- 安全设置：查看登录记录（如登录时间、登录设备、登录IP），异常登录提示，可手动退出所有登录设备。

### 3.2.3 内容管理功能

- 文章管理：新增文章、编辑文章、删除文章、恢复已删除文章（放入回收站，可手动清空），按发布状态（已发布、草稿）、发布时间、分类筛选文章，支持批量操作（批量删除、批量修改分类）；

- 分类管理：新增分类、编辑分类名称/描述、删除分类（删除前提示确认，避免误操作），调整分类排序，设置默认分类；

- 标签管理：新增标签、编辑标签名称、删除标签，支持标签合并、标签搜索，标签云显示控制（调整字体大小、显示数量）；

- 草稿管理：自动保存文章草稿（避免意外丢失），查看所有草稿，继续编辑草稿、删除草稿、发布草稿。

### 3.2.4 编辑器功能（核心创作工具）

- 基础排版：支持标题（H1-H6）、段落、加粗、斜体、下划线、删除线、引用、有序/无序列表、代码块（高亮显示，支持多种编程语言）；

- 媒体插入：插入本地图片、网络图片（支持图片预览、裁剪、删除），插入视频（支持网络视频链接）、附件（可选，如文档、压缩包）；

- 高级功能：支持表格插入/编辑、锚点设置、页面预览（预览发布后效果）、自动保存（每30秒自动保存一次）、草稿备份；

- 优化点：编辑器界面简洁，支持夜间模式，可自定义字体、行距，支持快捷键操作（提升创作效率）。

### 3.2.5 评论管理功能

- 评论查看：查看所有游客可见的评论（仅管理员可发布/回复评论），按评论时间、文章分类筛选评论；

- 评论操作：审核评论（因仅管理员可评论，可简化审核流程，仅需删除不当评论）、删除评论、回复评论（回复后游客可查看）；

- 优化点：评论列表清晰，区分管理员评论与游客可见评论（仅管理员可发布），支持关键词搜索评论。

### 3.2.6 数据统计功能

- 核心数据：文章总数量、总阅读量、总评论量（管理员发布/回复的评论）、总收藏量（管理员操作）；

- 明细数据：单篇文章阅读量、评论量、收藏量，按时间维度（日、周、月）统计阅读趋势，热门文章排行（按阅读量排序）；

- 展示形式：以图表（折线图、柱状图）形式直观展示数据，支持数据导出（可选，如Excel格式）；

- 优化点：数据实时更新，图表简洁易读，支持筛选时间范围查看数据。

### 3.2.7 系统设置功能

- 站点设置：修改博客名称、logo、域名（可选）、网站描述、版权信息，设置首页banner图、导航菜单（添加/删除/调整顺序）；

- 界面设置：设置博客主题（支持浅色/深色主题，可自定义配色）、字体样式、页面动画效果（控制微动效开关）；

- 安全设置：开启/关闭注册入口（默认关闭，仅本人可注册）、设置登录密码复杂度、开启异常登录提醒；

- 其他设置：设置文章默认排版、图片存储路径（本地/云存储）、评论显示规则（是否显示评论时间、管理员标识）。

# 四、界面需求（UI Requirement）

## 4.1 整体设计原则

- 风格：简洁高级、极简克制，突出文字质感，避免冗余元素，配色统一（建议主色调1-2种，辅助色1种，背景色以浅色为主，提升阅读舒适度）；

- 适配：响应式设计，PC端优先，移动端适配（调整排版、隐藏非必要模块，确保手机端阅读流畅）；

- 交互：微动效克制高级（如文章卡片hover效果、按钮点击反馈、页面加载动画），避免过度动画影响体验；

- 一致性：所有页面字体、间距、按钮样式、图标风格统一，提升整体质感。

## 4.2 核心页面界面要求

### 4.2.1 首页界面

- 顶部导航栏：固定在顶部，滚动时轻微收缩（提升高级感），包含logo（可点击返回首页）、导航菜单（首页、分类、标签、关于）、搜索框（右侧）、登录按钮（仅游客可见，管理员登录后显示个人中心入口）；

- Banner区：简洁大气，可设置一张背景图+简短文字（如博客slogan），支持管理员后台修改；

- 文章列表：卡片式布局，每张卡片包含文章标题、发布时间、分类、阅读量、简短摘要（可选）、封面图（可选），hover时轻微上浮/阴影加深，分页或滚动加载；

- 侧边栏（PC端）：显示分类导航、标签云、热门文章，移动端隐藏，可通过侧边滑出查看；

- 底部：简洁明了，包含版权信息、博主联系方式、网站运行时间（可选），管理员可后台编辑。

### 4.2.2 文章详情页界面

- 文章头部：标题（居中，字体加粗）、副标题（可选）、发布信息（发布时间、作者、分类、标签、阅读量、评论量、收藏量），排版整齐，间距适中；

- 文章内容：字体清晰（建议16px-18px），行间距1.8-2.0，段落间距适中，代码块单独高亮显示（区分于正文），图片居中显示，支持点击放大；

- 操作区：文章底部设置管理员操作按钮（收藏、分享、编辑、删除），游客无操作按钮；评论区标题、评论列表（区分管理员评论），管理员可在此发布/回复/删除评论；

- 相关推荐：文章底部显示同分类/同标签的3-5篇文章，卡片式布局，简洁易点击；

- 辅助功能：右侧固定返回顶部按钮，文章内容区可设置字体大小调节、夜间模式切换按钮。

### 4.2.3 管理员后台界面

- 布局：左侧侧边栏（导航菜单，包含内容管理、评论管理、数据统计、个人中心、系统设置），右侧内容区（对应功能页面），顶部显示登录状态、退出登录按钮；

- 风格：简洁高效，以功能性为主，配色与前台统一，避免花哨设计，操作按钮清晰可见（如新增、编辑、删除按钮）；

- 编辑器界面：全屏式编辑器，左侧编辑区，右侧预览区（可选，支持实时预览），顶部工具栏（排版、插入媒体等功能），底部保存/发布/草稿按钮。

# 五、非功能需求（Non-Functional Requirement）

## 5.1 性能需求

- 加载速度：PC端首页加载时间≤2秒，文章详情页加载时间≤1.5秒，后台操作响应时间≤1秒，避免卡顿；

- 图片加载：支持图片懒加载（滚动到可视区域再加载），压缩图片大小，提升加载速度；

- 稳定性：系统运行稳定，无频繁崩溃、报错，文章保存、评论操作无数据丢失。

## 5.2 安全需求

- 账号安全：管理员密码加密存储，登录失败3次后锁定账号（1小时后自动解锁，或管理员手动解锁），支持密码重置（仅本人可操作）；

- 数据安全：文章、评论、个人信息等数据定期备份（可手动备份+自动备份），防止数据丢失；

- 访问安全：禁止非法访问管理员后台，未登录状态下访问后台自动跳转至登录页；过滤恶意评论、恶意请求，防止SQL注入、XSS攻击。

## 5.3 易用性需求

- 操作简洁：管理员后台操作流程简单，创作、编辑、管理功能一步到位，无需复杂操作；

- 提示清晰：操作成功/失败、删除确认、密码错误等场景，给出明确提示，避免误操作；

- 学习成本低：界面直观，功能标识清晰，管理员无需专业技术，即可快速上手操作。

## 5.4 可扩展性需求

- 预留扩展接口：后续可扩展社交功能（如评论开放给游客、添加关注功能）、付费阅读、广告植入等功能；

- 主题扩展：支持自定义主题，管理员可自行更换博客风格，无需修改核心代码；

- 功能扩展：支持添加插件（如网站统计插件、防盗链插件），提升博客功能丰富度。

# 六、技术选型建议（可选）

## 6.1 前端技术

- 框架：React（灵活度高，适合后续扩展）；

- UI组件库：结合“高级感”需求，推荐Naive UI（Vue3）、ShadCN UI（React），简洁高级，适配博客风格；

- 辅助工具：Tailwind CSS（快速构建高级界面，响应式适配）、Markdown编辑器（如vue3-markdown-it、react-markdown，支持多格式排版）、Chart.js（数据统计图表）。

## 6.2 后端技术

- 框架：Java（SpringBoot，稳定性高，适合后续扩展）；

- 数据库：MySQL（关系型数据库，存储文章、评论、管理员信息等，易用性高）；
